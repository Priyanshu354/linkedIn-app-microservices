# LinkedIn-like Scalable Backend (Microservices Architecture)

A production-grade backend system inspired by LinkedIn, built using **Spring Boot microservices**, **event-driven architecture (Kafka)**, and **centralized JWT authentication at API Gateway**.

---

## Overview

This project demonstrates the design of a **distributed social networking backend**, supporting:

* 👤 User management & authentication
* 📝 Post creation & engagement (likes)
* 🤝 Connection system (graph-based relationships)
* 🔔 Event-driven notifications
* 🌐 API Gateway routing

The system is designed with **scalability, loose coupling, and fault tolerance** in mind.

---

## 🏗️ Architecture

### 🔹 Microservices

* **User Service** – Handles users & authentication (JWT generation)
* **Post Service** – Manages posts and likes
* **Connection Service** – Graph-based relationships (Neo4j)
* **Notification Service** – Kafka-based event processing
* **API Gateway** – Centralized routing & authentication
* **Discovery Server (Eureka)** – Service registration

---

## 🔐 Authentication & Security

* Implemented **JWT-based authentication at API Gateway**
* Tokens generated by User Service and validated at Gateway
* Gateway:

  * Validates JWT
  * Extracts user context
  * Forwards request to services

### 🔹 Benefits:

* Centralized security
* Stateless authentication
* Reduced duplication across services

---

## ⚡ Event-Driven Architecture (Kafka)

Integrated **Apache Kafka** for asynchronous communication between services.

### 🔹 Topics Implemented:

* `post-create-topic` → post creation events
* `post-like-topic` → like events
* `connection-sent-topic` → connection requests
* `connection-accept-topic` → connection acceptance
* `user-create-topic` → sync users to Neo4j

### 🔹 Consumers:

* **Notification Service** → handles user activity notifications
* **Connection Service** → processes graph updates

### 🔹 Kafka Configuration:

* 3 partitions
* 1 replica

### 🔹 Benefits:

* Loose coupling
* High scalability
* Asynchronous processing

  * Services

### 🔹 Kubernetes

* Deployments for services
* Services for networking
* Ingress for routing
* Kafka + UI (Kafbat) integration

---

## 🔄 System Flow

1. Client → API Gateway
2. Gateway validates JWT
3. Request routed to microservice
4. Services communicate via REST / Kafka
5. Events published to Kafka
6. Consumers process events (notifications, graph updates)
